# WebLarek

Клиентская часть интернет-магазина для веб-разработчиков.
Проект показывает каталог товаров, карточку предпросмотра, корзину и оформление заказа через API.

## Стек
- `TypeScript`
- `SCSS`
- `Vite`
- `HTML`

## Быстрый старт

```bash
npm install
npm run dev
```

## Сборка

```bash
npm run build
```

## Предпросмотр production-сборки

```bash
npm run preview
```

## Структура проекта

- `src/main.ts` — точка входа и оркестрация приложения
- `src/components/base/` — базовые классы (`Api`, `Component`, `EventEmitter`)
- `src/components/models/` — модели данных приложения
- `src/components/views/` — UI-компоненты и формы
- `src/types/index.ts` — общие интерфейсы и типы
- `src/utils/constants.ts` — константы API/CDN и маппинг категорий
- `src/utils/utils.ts` — вспомогательные функции для DOM и шаблонов
- `src/scss/styles.scss` — корневой файл стилей

## Архитектура

Проект построен по принципу разделения ответственности между тремя слоями:

- `Model` — хранение и изменение данных (каталог, корзина, покупатель)
- `View` — отображение интерфейса (карточки, модалка, формы, корзина)
- `Presenter / App orchestration` — связывание моделей и представлений через события

Коммуникация между слоями выполнена через `EventEmitter`, что упрощает масштабирование и поддержку.

## Основные сценарии

1. Загрузка каталога товаров с сервера.
2. Открытие карточки товара в модальном окне.
3. Добавление и удаление товаров из корзины.
4. Ввод данных покупателя (оплата, адрес, контакты).
5. Отправка заказа и отображение экрана успешного оформления.

## Работа с API

- Получение товаров: `GET /product`
- Отправка заказа: `POST /order`

Базовый адрес API формируется из переменной окружения `VITE_API_ORIGIN`.

## Примечания

- Валидация формы выполняется в `BuyerModel`.
- Состояние корзины и форм синхронизируется через события.
- UI обновляется реактивно при изменении данных в моделях.
